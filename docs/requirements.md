## 1. 외부차량 관리

### 1-1. 목적별 외부차량 검색

단지 내 방문한 모든 외부차량의 정보를 종류별로 검색·확인할 수 있어야 한다.

| 검색 구분 | 내용 |
|-----------|------|
| 방문차량 | 일반 방문 입차 차량 전체 조회 |
| 주차예약차량 | 사전 예약 등록 후 입차한 차량 조회 |

**필수 기능**
- 차량번호, 입차일시, 출차일시, 방문횟수, 방문목적 등 통합 조회
- 차량별 입차 방문 기록 및 상세 데이터 확인
- 목적별 필터링 및 기간 검색
- 조회 결과 인쇄/엑셀 출력

**현재 구현 상태**
- `GET /api/slots` — 슬롯별 차량 조회 ✅
- `GET /api/reservations` — 예약차량 조회 ✅
- 종류별 통합 검색 페이지 ❌ (미구현)

## 2. 블랙리스트 차량 관리

### 2-1. 개요

무분별한 예약차량, 미등록 차량, 장기불법차량 등을 블랙리스트로 등록하여 입차를 제한한다.

### 2-2. 동작 흐름

```
블랙리스트 차량 입차 시도
    → ANPR(번호인식 CCTV) 번호 확인
    → 블랙리스트 매칭
    → 차단기 미개방
    → 전체 경비근무자 스마트폰 즉시 알림
```

### 2-3. 블랙리스트 등록 항목

| 항목 | 비고 |
|------|------|
| 차량번호 | 필수 |
| 등록사유 | 불법주차, 미등록, 장기점거 등 |
| 등록일 | 자동 기록 |
| 처리담당자 | |

### 2-4. 예외 처리

- **긴급자동차** (998, 999로 시작): 블랙리스트 등록 여부와 무관하게 자동 입차 허용 (카메라 확인 후)

### 2-5. 알림 내용 예시

```
시큐원 콤미 관리자 오후 2:00
블랙리스트 차량입차
블랙리스트에 등록된 01하7979 차량이 입차하였습니다.
```

**현재 구현 상태**
- Blacklist 모델 ❌ (미구현)
- ANPR 웹훅 연동 ✅ (`/api/webhooks/anpr`)
- 블랙리스트 입차 알림 ❌ (미구현)

## 3. 입주민 보안관리 앱

> 입주민용 앱 (시큐원풀미). 현재 시스템의 모바일 앱(시뮬레이터)과 별개로, 실제 입주민이 사용하는 앱 기능 명세.

### 3-1. 주요 기능 5가지

#### ① 비상콜 시스템
- 앱 내 버튼 1회 클릭으로 비상 알림 발생
- 아파트 경비근무자 전원에게 즉시 호출 전송
- → [4. 비상콜 시스템](#4-비상콜-및-문자신고-시스템) 참조

#### ② 세대 방문차량 예약
- 입주민이 앱에서 방문 차량 번호판 사전 등록
- 등록된 차량은 ANPR 인식 후 차단기 자동 개방
- → [5. 방문주차 사전예약](#5-방문주차-사전예약) 참조

#### ③ 문자신고 및 신고이력 확인
- 입주민 → 경비근무자에게 민원·신고 문자 발송
- 경비근무자 처리 결과 즉시 확인 가능
- 신고 이력 전체 조회

#### ④ 유관기관 및 근무자 확인
- 인근 경찰서, 소방서 등 유관기관 연락처 조회
- 현재 근무 중인 경비근무자 연락처 확인
- 관리사무소에서만 근무자 근태현황(출·퇴근) 조회 가능

#### ⑤ 공지사항
- 단지 공지사항 확인
- 특별방범활동 기간 등 내용 전파

## 4. 비상콜 및 문자신고 시스템

### 4-1. 비상콜 기능

**흐름**
```
입주민 앱 → 비상콜 버튼 클릭
    → 경비근무자 앱에 긴급호출 알림
    → 경비근무자 확인 및 출동
    → 2차 대처 (세대 방문 또는 관련 기관 연락)
```

**세대 내 비상 상황 예시**
- 가스 누출, 화재, 침수 등 위급 상황
- 외출 중 세대 내부 문제 발생

### 4-2. 문자신고 기능

**흐름**
```
입주민 앱 → 문자신고 작성 및 전송
    → 경비근무자 앱에 신고내용 수신 알림
    → 경비근무자 처리 후 결과 회신
    → 입주민 처리결과 확인
```

**실제 사용 사례 (참고자료)**

| 구분 | 내용 |
|------|------|
| 문자신고 | "111동 인근입니다 주차단속 스티커 더 붙여주시고 계도 해주세요" |
| 문자신고 | "이중주차 시간이 지났는데 주차되어있습니다. 단속 확인해주세요 B109 114동 3-5라인 앞쪽입니다" |
| 문자신고 | "오후 1시 50분에도 주차중인데 강력스티커 처분 바랍니다" |

**관리자 화면 필요 기능**
- 신고 접수 목록 (날짜/유형/처리상태 필터)
- 담당자 배정 및 처리 상태 업데이트
- 처리 결과 회신 기능
- 이력 전체 조회 및 인쇄

**현재 구현 상태**
- Alert 모델 ✅ (DB 스키마)
- `GET /api/alerts` ✅
- 입주민 → 관리자 문자신고 플로우 ❌ (미구현)

## 5. 방문주차 사전예약

### 5-1. 예약 방식

| 방식 | 설명 |
|------|------|
| 횟수제 | 월 N회 예약 가능 (예: 10회/月) |
| 시간제 | 월 N시간 예약 가능 (예: 200시간/月) |

- 아파트 관리사무소에서 세대별 할당량 결정
- 월 부여된 예약 초과 시 추가 사용 불가 (아파트 결정에 따라 추가 가능)

### 5-2. 예약 프로세스

```
입주민 앱에서 방문차량 번호 + 입차예정시간 등록
    → 예약차량 DB 저장
    → 차량 입차 시 ANPR CCTV 번호 인식
    → 예약 차량 매칭 확인
    → 차단기 자동 개방
    → 입주민 스마트폰으로 "예약방문차량 입차" 알림 발송
```

**알림 예시**
```
시큐원 콤미 오후 3:22
예약방문차량입차
예약하신 12호8888(이준성) 차량이 입차하였습니다.
```

### 5-3. 초과사용 관리비 계산

- 월 예약 시간을 초과 사용한 세대에 대해 초과 시간만큼 관리비 부과 가능
- 단가 예: 분당 100원 (아파트 결정사항)
- 엑셀 출력 지원

**초과 사용 계산 예시 (참고자료)**

| 동/호 | 초과시간 | 금액 |
|-------|----------|------|
| 2819동 102호 | 10시간 | 9,600원 |
| 2820동 1503호 | 16시간 20분 | 15,680원 |
| 2820동 401호 | 121시간 | 116,160원 |

**현재 구현 상태**
- Reservation 모델 ✅
- `POST /api/reservations` ✅
- `GET /api/reservations` ✅
- 횟수/시간제 할당량 관리 ❌ (미구현)
- 초과사용 관리비 자동 계산 ❌ (미구현)
- ANPR 연동 자동 차단기 개방 ❌ (미구현)

## 6. 불법주차 단속 시스템

### 6-1. 단속 앱 기능 (경비근무자용)

차량번호 촬영 시 즉시 아래 4가지 구분으로 확인:

| 구분 | 내용 |
|------|------|
| 등록차량 | 입주민 정보 확인 |
| 미등록차량 | 미등록 차량으로 표시 |
| 방문예약차량 | 예약 정보 확인 |
| 방문차량 | 방문 기록 확인 |

### 6-2. 단속 문자 발송 유형

| 유형 | 발송 시점 |
|------|-----------|
| 이동주차 문자 | 1차 단속 시 — 즉시 차량 정보 확인 후 이동주차 요청 |
| 강력단속 문자 | 2차 단속 시 — 이동 미조치 후 강력 단속 예고 |
| 스티커부착 문자 | 스티커 부착 차량에 대한 안내 |

> 문자 발송은 통록차량(연락처 보유) 기준. 개인정보 보호법 준수 계약 체결 후 운영.

### 6-3. 단속 시간대 계획

| 구분 | 주간 | 야간 |
|------|------|------|
| 운영시간 | 09:00~10:00 / 13:00~14:00 / 15:30~16:30 | 아파트 상황에 맞게 운영 |
| 단속횟수 | 주간 3회 (상황에 따라 변경) | - |

**1차 순찰 중점사항**
- 지하주차장 전 지역 위반차량 확인 운영
- 지하 통로주차 강력 단속 (강력스티커 부착)
- 아파트 운영방식에 따라 발견시 즉각 강력스티커 부착 운영

**2~3차 순찰 중점사항**
- 발견된 불법주차차량에 대한 이동 및 조치 완료여부 확인
- 미이동 통계 도로 미이동차량에 대한 강력 단속 관리
- 지하 주차위치 위반차량에 대한 단속관리 강화 운영 실시

**단속 절차**
```
이동안내문(아파트 협의) → 경고장 발부(미이동시 연락 및 문자 운영)
단속일시 및 단속내용을 당사 시스템을 통해 관리운영 실시
```

### 6-4. 단속 강화 구역

- 주차금지 구역
- 장애인 주차구역 (장애인 외 주차 단속)
- 전기자동차 충전 주차구역 (비 EV 차량 단속)
- 불법주차 다수 발생지역 (통로주차 등)

### 6-5. 보고서 관리

- 불법주차 단속확인 보고서 자동 생성 및 인쇄
- 근무자별 단속관리 횟수 확인 (순찰 수 / 등록 차량 / 미등록 차량)
- 차량별 단속관리 횟수 및 상세 위반내용 조회
- 서버 전송으로 관리사무소에서도 동시 확인 가능

**현재 구현 상태**
- ANPR 웹훅 수신 ✅ (`/api/webhooks/anpr`)
- 블랙리스트/불법주차 단속 앱 기능 ❌ (미구현)
- 단속 문자 자동 발송 ❌ (미구현)
- 단속 보고서 생성 ❌ (미구현)

## 7. 순찰관리 시스템

### 7-1. 개요

단순 코퍼스(순찰기기) 운영에서 스마트폰 기반 순찰관리 시스템으로 전환.

### 7-2. 순찰 프로세스

```
순찰 구역 QR코드 스캔
    → 순찰 및 현장 사진 촬영
    → 특이사항 작성
    → 서버 전송
    → 관리자 점검 내용 및 통계 확인
```

### 7-3. 순찰 그룹 설정

| 설정 항목 | 내용 |
|-----------|------|
| 순찰그룹 설정 | 지상순찰 / 지하순찰 / 옥상순찰 등 목적별 그룹 |
| 순찰위치 편성 | 그룹별 순찰위치 상세 확인 |
| 순찰 순서 지정 | 그룹별 순찰 순서 확인 |

### 7-4. 순찰 보고서 기능

- 경비근무자별 실시간 순찰확인 가능 (시작/종료 확인)
- 순찰결과 서버 DATA 저장 및 확인
- 관리주체·본사 등 순찰결과 동시 공유
- 물자절약: 인쇄 없이도 결과 확인 가능
- 노후화에 따른 수리 불필요 (개인 스마트폰 활용)

### 7-5. 순찰결과 보고서 항목 (참고자료)

| 항목 | 내용 |
|------|------|
| 순찰지역 수 | 전체 순찰 구역 수 |
| 미순찰 | 순찰하지 않은 구역 수 |
| 확인 달성 | 실제 순찰 완료 수 |
| 확인 달성 지역 | 달성/미달성 지역 목록 |

**현재 구현 상태**
- 순찰 관련 기능 전체 ❌ (미구현)
